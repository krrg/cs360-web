Lab: Web Application Vulnerabilities

In this lab you will learn how to conduct and protect against two common attacks on web servers -- cross-site scripting (XSS) and corss-site request forgery (XSRF).

Gruyere

A good way to learn about these two attacks is to use the Google Code project called Gruyere. This project will walk you through several different vulnerabilities, showing you both how to exploit the vulnerability with a certain kind of attack, as well as how to create a secure server.

Read through the first few sections of the lab, dealing with XSS and XSRF. Carry out these attacks on the Gruyere web application and be sure you understand how they work.

Attack!

Use the Django list demo application we built in class to demonstrate both an XSS attack and an XSRF attack. Note, because Django comes with some built-in protections against these attacks, you may need to disable its security precautions to get the attacks to work. See for example these pages:

  • security in Django
  • autoescape in Django templates
  • CSRF protection in Django

Report

Write a report in the form of a tutorial, explaining how to exploit the XSS and XSRF vulnerabilities in the Django list demo application, as well as how to fix or avoid them. Your report should have the following organization:

  1. Introduction: Short introduction to the tutorial (goals, what the reader needs to know before starting the tutorial).
  2. XSS Attack: Explain what an XSS attack is. Show how you can make this attack work on the list application, walking through all the steps required. Then explain how to protect against this attack, again walking carefully through all the steps. Include screenshots for both the case where the attack works and where the attack doesn't work.
  3. XSRF Attack: Explain what an XSRF attack is. Show how you can make this attack work on the list application, walking through all the steps required. Then explain how to protect against this attack, again walking carefully through all the steps. Include screenshots for both the case where the attack works and where the attack doesn't work.

Precautions

Because this is a security lab, some precautions are in order:

  • You should never attack a site unless you have permission of the owner.
  • You should not use what you learn in this lab to attack any applications other than your own.
  • Before conducting an attack, always get permission from the appropriate authority or organization, for example your company's security team.

Submission

Turn in PDF of your report using learningsuite.byu.edu.
